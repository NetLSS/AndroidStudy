## 화면에 그려지는 디자인 요소 위젯

- 위젯 이란?
    - 버튼, 텍스트뷰, 이미지뷰와 같이 화면에 그려지는 디자인 요소

<br/>

- 디자인 탭의 팔레트에 있는 탭을 기준으로 설명 

### Text

- 글자를 화면에 나타내거나 입력받을 수 있는 위젯들
- 디자인 탭에서..
    - Ab 아이콘에 언더바가 있는 위젯은 모두 EditText 위젯

### Buttons

- 사용자로부터 클릭 또는 터치 관련 이벤트를 받을 수 있는 위젯들
- 버튼, 라디오 버튼, 체크 박스, 스위치 등
- 클릭 vs 터치
    - 터치 : 손을 데는 순간 발생
    - 클릭 : 터치 후 같은 위치에서 손을 떼었을 때

### Widgets

- 이미지, 웹 사이트, 별점 표시, 진행 상태 등의 정보를 화면에 그리는 위젯 모음

---

## 2.2. 텍스트 뷰

- 화면에 텍스트를 출력하는 가장 기본적인 위젯
- text 속성에 직접 입력하는 것 보다는 strings.xml 에 미리 사용하는 것 권장
    - 다국어 처리
    - 텍스트 수정 등 관리 용이
- 몰랐던 것: 입력 필드가 `@`으로 시작되면 특정 리소스를 참조하고 있다는 의미
    - ex) `@string/name_str`

**textColor 로 텍스트 색상 지정**

- RGB(+투명)
- 16진수(0~F)로 표현
- `#투명_Red_Green_Blue`
- 투명은 입력하지 않으면 FF(불투명), (00에 가까울 수록 투명)
- 색상도 문자열과 같이 직접 입력보다는 colors.xml 에 작성 권장

**textSize 로 텍스트 크기 지정**

- 텍스트가 입력되는 위젯에서는 주로 sp를 사용
- sp?
    - Scale-independent Pixels 의 약자 (화면 스케일에 독립적으로 크기를 조절할 수 있음)
    - 문자열 크기를 나타내기 위해 사용
    - dp 말고 sp를 쓰는 이유?
        - 같은 해상도에서 문자열의 크기를 다르게 사용하는 경우가 있기 때문
        - 눈이 안좋은 사람이 확대할 때 다른 위젯 영향없이 글씨만 커지거나 작아지게 할 수 있음
- dp?
    - Device Independence Pixel
- 텍스트 사이즈 또한 dimens.xml 에 따로 입력 후 사용 권장 (파일 생성 필요)
    - ```xml
      <dimen name="단위이름">150sp</dimen>
      ```

**textStyle 텍스트 스타일 지정**

- 중복 적용 가능

**입력 가능한 줄 수 설정 : maxLines, minLines**

- maxLines : 입력가능한 최대 줄 수
- minLines : 입력 가능한 최소 줄  

**텍스트뷰 한 줄로 보이기 : singleLine**

- 텍스트 뷰를 한줄로 보이게
- maxLine 을 1로 주는 것 과는 다른 점
    - 여러 줄이 있을 때 두번 째 줄을 없애지 않고 줄 사이 개행을 없애 한줄로 보이게 함

**알줄임 표시하기 : ellipsize**

- maxLines 속성이 1이거나 문자열이 길어서 잘릴 때 설정
- 말줄임(...)표시를 하거나 , marquee로 글자를 좌우로 움직이게 할 수 있음
- 속성
    - none : 설정 안함
    - start: 첫 부분을 말줄임 표시
    - middle: 중간 부분을 말줄임 표시
    - end: 끝 부분을 말줄임 표시
    - marquee: 글자가 흐르는 효과를 줌. (singleLine을 true로 주어야함!) (maxLines 1로 주어도 되는데 충돌 가능성, marquee 기능과 충돌 가능성), 텍스트가 포커스를 받아야 하므로 focusable 속성 을 auto, focusablenTouchMode 속성을 ture로 설정. -> 전광판 처럼 텍스트가 움직임

**글꼴 지정하기 : fontFamily**

- 기본 및 외부 폰트 지정 가능.

**비율로 글꼴 크기 지정하기: emd**

- 텍스트뷰의 크기를 나타낼 때 현재 글꼴의 크기를 기준으로 설정하는 상댓값.
- 글꼴 크기에 텍스트뷰를 맞춰 글꼴이 커질 때 텍스트와 텍스트뷰의 비율을 유지

**텍스트뷰 높이 고정하기 : lines**

- 지정한 높이로 높이를 항상 고정 (더 늘어나지 않음)

**텍스트 전체 길이 제한하기 : maxLength**

- 지정 길이만큼만 보이고 나머지는 보이지 않게.

## 2.3 EditText

- 주로 입력을 받는 용도

**클릭하면 사라지는 미리보기 : hint**

- place holder 라고도 불림

**입력 형식 설정 : inputType**

- 옵션에 다라 키보드 모양을 달리 보여줌
- 입력 제한
- number, textPassword 등

**이벤트 설정 : imeOptions**

- ime(input method editor)=텍스트 편집기
- 입력이 완료된 상황에서 다름 이벤트로 어떤 처리를 할지.
- options
    - normal : 특별한 기능 없음
    - actionUnspecified : 특별한 액션 없음
    - actionNone : 액션을 사용하지 않음
    - actionGo : 어딘가로 넘어감. (url 입력 후 해당 페이지로 넘어가기 등)
    - actionSearch : 검색
    - actionSend : 메일, 메세지 보내기
    - actionNext : 다음으로 넘어가기, 디음 입력창
    - actionDone : 입력 완료, 키보드 숨김
    - actionPrevious : 이전 단계로 돌아가기, 이전 입력창으로 이동

## 2.4 이미지 버튼

- 텍스트뷰를 상속 받아. 텍스트뷰와 비슷
- 버튼, 이미지 버튼 둘다 백그라운드 속성으로 이미지를 줄 수 있음
  - 버튼은 백그라운드 이미지 위에 텍스트만
  - 이미지 버튼은 백그라운드 이미지 위에 아이콘과 같은 이미지를 추가할 수 있음
  - 차이점 : 클릭을 받아주는 리스너를 텍스트에 구현했느냐, 이미지뷰에 구현했느냐의 정도임
  - 버튼은 텍스트뷰의 속성을, 이미지 버튼은 이미지 뷰의 속성을 거의 그대로 사용

**이미지 사용하기**

- png 추가 후 src 속성에 적용

**투명 배경 설정하기**

- background 속성으로 `@android:color/transparent` 적용
  - 기본 회색 배경이 없어짐 

**이미지 크기 설정하기 : scaleType**

- matrix : 실제 이미지를 좌측 상단부터 이미지버튼 크기만큼만 보여줌
- fitXY : 상하좌우를 이미지뷰 또는 이미지 버튼 크기에 맞게 늘려줌
- fitStart : 좌측 상단부터 시작해서 비율에 맞게 이미지 크기를 조절해서 위젯 안에 채워줌
- fitCenter : 중앙을 기준으로 비율에 맞게 이미지 크기를 조절하여 위젯 안에 채워줌
- fitEnd : 우측 하단부터 시작해서 비율에 맞게 이미지 크기를 조절하여 위젯 안에 채워줌
- center : 실제 이미지 사이즈 대로 정중앙에 위치. 이미지가 크면 위아래가 잘림
- centerCrop : 가로세로 사이즈 중 근접한 길이를 기준으로 나머지 한쪽을 잘라 비율을 맞춤. 뷰에 이미지를 가득 채워주기 때문에 앨범 이미지를 섬네일로 보여줄 때 많이 사용
- centerInsize : 이미지가 위젯보다 크면 fitCenter와 동일하게 동작하고 작으면 위젯의 중앙에 위치시킴

**이미지 영역에 색 채우기 : tint**

- 이미지 영역에 색을 채우는 속성
- 이미지의 투명도를 기준으로 색이 적용 된다.
- 일반적으로 투명 배경을 가진이미지에 사용한다.

**투명도 조절하기 : alpha**

- 1부터 0까지 값을 입력
- 1이면 불투명, 0이면 투명

## 2.5 라디오 그룹과 라디오 버튼

- 여러개의 선택지 중 하나만 선택할 때
- 단독으로 사용할 수도 있지만, 라디오 그룹과 함께 사용하면 다루기가 더 쉬움
- `setOnChecked...` 등 으로 변경 처리

참고
- 레이아웃 파일에 id를 입력하면 모든 아이디에 R.id. 이라는 일종의 접두어가 붙어서 코드로 생성됨
- 안드로이드가 리소스를 관리하는 R이라는 클래스를 생성하고, 그 안에 리소스 아이디를 관리하는 id 클래스가 추가되는 것
- 그룹으로 관리하면 그룹안에 있는 라디오 버튼 처리를 간단하게 할 수 있음

**라디오 버튼 배치하기 : orientation**

- 라디오 그룹은 리니어 레이아웃에 라디오버튼을 담을 수 있는 형태의 레이아웃임
- 가로, 세로 정렬 선택

**선택된 라디오버튼 설정하기 : checkedButton**

- 미리 선택되어있는 라디로 버튼 설정

## 2.6 체크박스

- 여러개의 선택지가 있을 때에도 하나가 아닌 여러개를 선택할 때
- 기본적으로 1개의 위젯당 1개의 리스너
- 또는 공통으로 사용되는 리스너 1개만 구현 해서 사용 (when문 활용, 전달 되는 뷰의 아이디로 구분)
- 컴파운드 버튼
  - 체크박스는 컴파운드 버튼을 상속 받아서 만들어짐
